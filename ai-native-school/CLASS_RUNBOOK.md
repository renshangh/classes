# Live Class Runbook (Epic A-1)

This runbook defines the standard operating procedures for live class sessions.

## 1. Pre-Class Checklist (T-minus 30 mins)
- [ ] **Technical Check**: Test camera, microphone, and screen sharing.
- [ ] **Materials Check**: Ensure Lesson `README.md` and `handout.md` are finalized on `develop`.
- [ ] **Telegram Announcement**: Post the welcome message and Google Meet link to the class group.
- [ ] **Google Drive**: Ensure the `Recordings` folder for the session is ready.

## 2. During Class
- [ ] **Start Recording**: Start the Google Meet recording as soon as the lecture begins.
- [ ] **Delivery**: Send bilingual content blocks (stacked format) to Telegram as they are covered.
- [ ] **Checkpoint Polls**: Send a poll after every major concept (understood/need more detail/stuck).
- [ ] **Triage**: If a student uses the "I'm Stuck" button, acknowledge immediately and triage according to Epic A-4.

## 3. Post-Class Wrap-up
- [ ] **Homework Announcement**: Push the `exercise.md` instructions and rubric link to Telegram.
- [ ] **Recording Upload**: Move the Google Meet recording to the `OpenClaw Classroom/Recordings` folder.
- [ ] **Issue Review**: Review any blockers raised during class and update the GitHub Project board.
- [ ] **Attendance Summary**: Generate a summary of participation from the bot's poll data.
