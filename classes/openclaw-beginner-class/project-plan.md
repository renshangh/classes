# Project Plan: OpenClaw Beginner Program (Prep + 3 Classes)

## Why this update
Based on review feedback, we are shifting from a "content-only class package" to a full **program rollout**.

Key additions:
- Add a **Prep Class** before Class 1
- Cover logistics, device/access readiness, and delivery operations
- Increase AI-assisted learning elements throughout the program
- Break delivery into clear tasks for multi-agent execution

## Program structure
- **Prep Class (new):** setup + logistics + readiness validation
- **Class 1:** Foundations + first practical win
- **Class 2:** Multi-step workflows + tool use
- **Class 3:** Agentic design + capstone

## Workstreams and tasks

### A) Program Design
- A1. Finalize session sequence: Prep + 3 classes
- A2. Define outcomes and success criteria per session
- A3. Define assignment cadence and feedback rhythm

### B) Delivery Infrastructure
- B1. Select live platform (Zoom/Meet) + backup method
- B2. Define Telegram operating model (announcements, Q&A, homework help)
- B3. Create onboarding checklist and pre-class readiness form

### C) AI-First Learning Experience
- C1. Add prompt templates for every session
- C2. Add stuck-support protocol (hint ladder + escalation)
- C3. Add process-focused rubric (not output-only)

### D) Repo and Execution Management
- D1. Structure repo for course scaling
- D2. Create issue/task board and ownership model
- D3. Enforce PR-first review gates before merge/release

## Roles (initial)
- **Sarah (lead):** A1, A2, C1, C2, D1, D3
- **Aether:** B1, B2, B3, C3, D2
- **Randy (owner):** approve tooling/platform decisions and final curriculum acceptance

## Milestones
1. **M1: Program baseline approved**
   - Prep class + class plans + support model documented
2. **M2: Delivery setup approved**
   - live platform, Telegram process, and onboarding validated
3. **M3: Task execution started**
   - issues assigned, PR workflow active, incremental reviews
4. **M4: Release candidate**
   - all materials finalized for first teaching cohort

## Review gates
- Gate 1: Program structure approval
- Gate 2: Delivery/logistics approval
- Gate 3: Content quality + hands-on flow approval
- Gate 4: Final release approval
