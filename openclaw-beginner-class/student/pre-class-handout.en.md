# Pre-Class Handout (Detailed Readiness)

## A) OpenClaw Verification Checklist (Pass/Fail)
### 1. Device baseline
- [ ] Terminal opens and runs `pwd`
- [ ] At least 3GB free disk space
- [ ] Stable internet available

### 2. OpenClaw runtime
- [ ] `openclaw --version` works
- [ ] `openclaw status` works
- [ ] `openclaw gateway status` shows running (or student can start it)

### 3. Model access
- [ ] Student can send one prompt and receive a reply
- [ ] No auth / model permission errors

### 4. Telegram channel
- [ ] Student joined class Telegram group
- [ ] Student can mention bot and receive a reply

### 5. GitHub readiness
- [ ] GitHub account available
- [ ] Student can open class repo and PR link

### 6. Live-class readiness
- [ ] Camera/mic tested
- [ ] Can share screen

### 7. Final test
- [ ] Student completes: create folder + README task and confirms output

## B) Model Access Matrix
- **Option A (Recommended): Bring your own API/subscription**
  - Pros: independent, reliable, flexible
  - Risk: personal billing/limits
- **Option B: Class-provided shared access**
  - Pros: easy onboarding
  - Risk: quotas/concurrency bottlenecks
- **Option C: Observer mode (no direct execution)**
  - Pros: can still follow learning
  - Risk: reduced hands-on outcomes
