# 课前讲义（详细准备清单）

## A) OpenClaw 验证清单（通过/不通过）
### 1. 设备基础
- [ ] 能打开终端并运行 `pwd`
- [ ] 至少 3GB 可用磁盘空间
- [ ] 网络稳定

### 2. OpenClaw 运行状态
- [ ] `openclaw --version` 可执行
- [ ] `openclaw status` 可执行
- [ ] `openclaw gateway status` 显示运行中（或学员会启动）

### 3. 模型访问
- [ ] 能发送一次提示词并收到回复
- [ ] 无鉴权/权限报错

### 4. Telegram 课堂通道
- [ ] 已加入课程 Telegram 群
- [ ] 能 @ 机器人并收到回复

### 5. GitHub 就绪
- [ ] 已有 GitHub 账号
- [ ] 能打开课程仓库与 PR 链接

### 6. 直播课就绪
- [ ] 摄像头/麦克风测试通过
- [ ] 能进行屏幕共享

### 7. 最终测试
- [ ] 完成“创建文件夹 + README”任务并确认产出

## B) 模型访问方案矩阵
- **方案 A（推荐）：自带 API/订阅**
  - 优点：独立、稳定、灵活
  - 风险：个人费用与配额限制
- **方案 B：课堂统一提供访问**
  - 优点：上手快
  - 风险：共享额度/并发瓶颈
- **方案 C：旁听模式（不直接执行）**
  - 优点：可跟学流程
  - 风险：实操效果较弱
